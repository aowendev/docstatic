# このプロジェクトでClaudeを使用する

[Claude Code](https://claude.ai/code)はAnthropicのAI搭載CLIです。このプロジェクトでは、特定のタスクの開発アシスタントとして使用されています。docStaticのアーキテクチャの設計や開発には使用されていません。

## 手作業の自動化

Claudeは、手作業が必要となる反復的またはスクリプト化された作業を処理します。たとえば、多数のファイルにわたるフロントマターフィールドの一括更新、新しい構造に合わせたコンテンツの再フォーマット、既知のパターンに従ったボイラープレートの生成などです。これらは、作業の形が明確だが実行が煩雑なタスクです。

## Reactコンポーネントとダッシュボードの開発

ClaudeはReactコンポーネントやデータダッシュボードの作成と改善に使用されています。これには、新しいUIコンポーネントの実装、レイアウトとスタイリングの調整、TinaCMS GraphQL APIからのデータの接続が含まれます。Claudeは新しい抽象化を導入するのではなく、プロジェクト内の既存のコンポーネントパターンに従って作業します。

## コンテンツの健全性チェック

Claudeは、事実の一貫性、構造的なギャップ、論理的な流れについてドキュメントをレビューします。これは軽量なチェックであり、正式な編集レビューを置き換えるものではありませんが、コンテンツがレビュアーに届く前に明らかな問題を発見するのに役立ちます。

## バグの追跡

Claudeは、関連するコードを読み込み、最近の変更を確認し、原因を推論することで、バグの原因を絞り込むのに役立ちます。複数のファイルにまたがるバグや、設定と動作のクロスリファレンスが必要なバグに特に有用です。

## スタイルルールの作成

ClaudeはdocStaticスタイルセットのValeルールを作成するために使用されています。これには、新しいルールの作成、サンプルコンテンツへのテスト、誤検出を減らすためのパターンの改善が含まれます。ルールはドキュメントと並んで `.vale/styles/docStatic/` にコミットされます。

## 情報アーキテクチャの維持

Claudeはドキュメントセットの情報アーキテクチャの維持を支援します。タクソノミー、タグ、ページ構造をレビューして、一貫した分類と論理的なナビゲーションを確保します。これはコンテンツレビューと並行して実行される継続的なタスクです。

## ドキュメントの草稿作成

Claudeがドキュメントページの初稿を作成し、その後人間の著者がレビューして編集することがあります。草稿は出発点として扱われ、完成したコンテンツとは見なされません。

  ClaudeはdocStaticの設計やアーキテクチャには関与していません。その使用は、このページに記載されている開発およびコンテンツタスクに限定されています。
</Admonition>
