# タクソノミー

情報量が増えると、必要なものを見つけることが難しくなります。Markdown ではコンテンツにタグを付けることができますが、一貫性のチェック機能はありません。docStatic ではタクソノミーを使用して、コンテンツに[情報アーキテクチャ](https://en.wikipedia.org/wiki/Information_architecture)を適用できます。

## 情報アーキテクチャの設計

情報アーキテクチャを詳細にカバーするのはこのドキュメントの範囲を超えますが、以下にいくつかの提案を示します：

* 階層を3レベルまでに制限してください。これは従来の[インデックス作成](/docs/tags)によく対応しています。
* 用語を正規化してください。単数形や動詞よりも複数形を優先してください。フレーズよりも単語を優先してください。
* 頭字語や略語は省略せずに表記してください。
* トップレベルを10カテゴリ以内に制限するよう努めてください。
* 中間レベルを7カテゴリ以内に制限するよう努めてください。
* 葉レベルのカテゴリは必要な数だけ使用してください。

## フラットなタグシステムへの階層の適用

理想的には、すべての情報プラットフォーム（バグ追跡システム、wiki など）に情報アーキテクチャを適用すべきです。これは、タグが最も制約の厳しい基準を満たす必要があることを意味します：

* タグは、アンダースコア（\_）で区切られた親カテゴリを含める必要があります。
* 小文字のみ使用します。数字、スペース（代わりにハイフンを使用）、特殊文字は使用できません。

タグが不必要に長くなるのを防ぐため、ネストは最大3レベルの深さに制限されています。場合によっては、構造をフラット化する必要があります。たとえば、`regions_europe_uk_scotland` は論理的ですが、4レベルあるため `regions_uk_scotland` として構造化する必要があります。

## タクソノミータグの管理

CMS でコンテンツにタグを付ける際は、自分で定義したタグのセットに限定されます。

1. コレクションから Taxonomies を選択し、タクソノミーを追加します。
2. 中間レベルの子タグを追加します。これらには親タグを含める必要があります（parent\_child）。
3. 葉レベルの子タグを追加します。これらには親タグを含める必要があります（top\_mid\_leaf）。

<Admonition type="note" title="注意">
  子タグに親タグとアンダースコアを含める必要はありませんが、省略するとトピックにタグを追加する際にフラットなタグのリストが表示されます。
</Admonition>

## コンテンツへのタグ付け

CMS でトピックを編集する際、Search Mode を使用して用語を検索するか、Tree View から選択することができます。タグを追加すると、すべての親タグが自動的に追加されます。ただし、各タグのインスタンスは1つのみ追加されます。

<Admonition type="tip" title="ヒント">
  CMS の外でコンテンツにタグを付けることは避けてください。これにより、タグの一貫性が保たれ、コンテンツの検索が容易になります。
</Admonition>
