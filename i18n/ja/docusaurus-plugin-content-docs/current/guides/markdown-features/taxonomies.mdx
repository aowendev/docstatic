# Taxonomy tags

タクソノミータグを使用すると、サイトコンテンツに[情報アーキテクチャ](https://en.wikipedia.org/wiki/Information_architecture)を適用できます。バグ追跡システムやウィキなど、使用するすべてのドキュメントプラットフォームに適用する必要があります。最も制約の厳しいタグ要件として、スペースなしで小文字とアンダースコア（`_`）のみをサポートするフラット（非階層型）システムがあります。

docStaticはタクソノミータグに3階層の階層構造を適用します。タグが不合理に長くなるのを防ぐため、ネストは最大3階層に制限されています。場合によっては、構造をフラット化する必要があります（たとえば、`regions_europe_uk_scotland` は論理的ですが4階層になるため、`regions_uk_scotland` と構造化する必要があります）。

3階層の情報アーキテクチャを設計する場合、トップレベルカテゴリを約10個、中間レベルカテゴリを約7個、必要なだけのリーフレベルカテゴリを持つことが推奨されます。頭字語や略語は省略せずに記述し、冗長性を避けることを目指してください。

## タクソノミータグの編集

1. **Taxonomies** コレクションから、タクソノミーを **追加** します。
2. 小文字のみを使用して **タグ** を設定します。
3. **子要素** を追加し、親タグの後にアンダースコアを付けてタグを設定します。
4. 各子要素について、さらに子要素を追加します。

<Admonition type="note" title="注意">
  前のタグ名とアンダースコアなしでタグを入力することもできますが、その場合にトピックに追加すると、フラットリストとして表示されます。
</Admonition>
