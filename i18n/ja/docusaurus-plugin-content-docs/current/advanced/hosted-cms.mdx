# ホスト型 CMS

docStatic の2つのコアコンポーネントのうちの1つが CMS です（もう1つは SSG）。これまで、おそらくコマンドラインから `npm run dev` または `yarn dev` で起動するローカルバージョンのみを使用してきたと思います。しかし、docStatic はホスト型 CMS を使用する2つの方法もサポートしています。

## TinaCloud

最も簡単なホスト型 CMS オプションは [TinaCloud](https://tina.io/docs/tinacloud) です。無料アカウントに[登録](https://app.tina.io/register)し、GitHub リポジトリに接続してすぐに使い始めることができます。無料プランは最大2ユーザーをサポートしており、単独のライターや小規模なドキュメントチームには十分かもしれません。さらに多くのユーザーが必要な場合、Tina はさまざまな予算に合わせた[料金プラン](https://tina.io/pricing)を提供しています。

## セルフホスト

TinaCMS はオープンソースであるため、[セルフホスティング](https://tina.io/docs/self-hosted/overview)という選択肢もあります。この場合、サブスクリプション料金は発生しませんが、Tina インスタンスのセットアップには開発チームが必要で、サポートは自前で提供する必要があります。
