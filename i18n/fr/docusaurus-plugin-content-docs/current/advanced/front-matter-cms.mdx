Lors de la création de contenu dans un IDE tel que [VSCodium](https://vscodium.com/), les fonctionnalités fournies par le CMS ne sont pas disponibles. Dans un flux de travail typique, les développeurs créent ou modifient le contenu en modifiant directement le fichier MDX dans l'IDE, tandis que les rédacteurs nettoient et étiquettent le contenu dans le CMS. Si le développeur oublie d'inclure une date de dernière modification, il faut alors consulter les propriétés du fichier pour obtenir cette information. Il existe toutefois un CMS léger disponible sous la forme d'un plugin appelé [Front Matter](https://frontmatter.codes/). Le référentiel DocStatic contient un exemple de configuration de Front Matter. Pour plus d'informations, consultez la [documentation](https://frontmatter.codes/docs) Front Matter.
