# Vérification stylistique

L'une des fonctionnalités qui manque le plus aux rédacteurs travaillant dans le domaine du « docs-as-code » est un bon correcteur orthographique, grammatical et stylistique. Par défaut, la vérification orthographique est assurée par votre navigateur web ou votre environnement de développement intégré. Cela peut suffire pour les sites web simples, mais si vous avez besoin de quelque chose de plus avancé, vous aurez besoin d'un correcteur grammatical dédié.

## LanguageTool

DocStatic est testé avec [LanguageTool](https://languagetool.org/), un correcteur grammatical basé sur l'IA qui prend en charge plus de 20 langues. L'inscription au forfait de base est gratuite [ici](https://languagetool.org/register). Le [forfait premium](https://languagetool.org/premium) ajoute la prise en charge des guides de style pour les particuliers et les équipes. Des forfaits sur mesure pour les [entreprises](https://languagetool.org/sales) sont disponibles. Il existe également une [réduction](https://languagetool.org/premium) pour les étudiants et les organisations à but non lucratif.

## Auto-hébergement

LanguageTool étant open source, vous avez la possibilité de l'auto-héberger (https\://languagetool.org/dev). Dans ce cas, il n'y a pas de frais d'abonnement, mais vous aurez besoin d'une équipe de développement pour configurer votre instance LanguageTool et vous devrez assurer vous-même le support.
