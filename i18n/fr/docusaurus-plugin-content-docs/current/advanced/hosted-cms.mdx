# CMS hébergé

L'un des deux composants principaux de DocStatic est le CMS (l'autre étant le SSG). Jusqu'à présent, vous n'avez probablement utilisé que la version locale, invoquée à partir de la ligne de commande avec `yarn dev`. Mais DocStatic prend également en charge deux façons d'utiliser une version hébergée du CMS.

## TinaCloud

L'option CMS hébergé la plus simple est [TinaCloud](https://tina.io/docs/tinacloud). Vous pouvez [vous inscrire](https://app.tina.io/register) pour obtenir un compte gratuit, le connecter à votre dépôt GitHub et commencer à l'utiliser immédiatement. La version gratuite prend en charge jusqu'à deux utilisateurs, ce qui peut être suffisant pour les rédacteurs indépendants et les petites équipes de documentation. Si vous avez besoin de plus d'utilisateurs, Tina propose différents [forfaits tarifaires](https://tina.io/pricing) adaptés à tous les budgets.

## Auto-hébergé

TinaCMS étant open source, vous avez la possibilité de l'[auto-héberger](https://tina.io/docs/self-hosted/overview). Dans ce cas, il n'y a pas de frais d'abonnement, mais vous aurez besoin d'une équipe de développement pour configurer votre instance Tina et vous devrez assurer vous-même le support technique.
