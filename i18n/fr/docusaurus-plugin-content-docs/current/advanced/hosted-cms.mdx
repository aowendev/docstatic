# CMS hébergé

L'un des deux composants principaux de DocStatic est le CMS (l'autre étant le SSG). Jusqu'à présent, vous n'avez probablement utilisé que la version locale, invoquée à partir de la ligne de commande avec `yarn dev`. Mais, DocStatic supporte également deux façons d'utiliser une version hébergée du CMS.

## TinaCloud

L'option de CMS hébergé la plus simple est [TinaCloud](https://tina.io/docs/tinacloud). Vous pouvez [vous inscrire](https://app.tina.io/register) pour un compte gratuit, le connecter à votre dépôt GitHub et commencer à l'utiliser immédiatement. La version gratuite prend en charge jusqu'à deux utilisateurs, ce qui peut être suffisant pour les rédacteurs isolés et les petites équipes de documentation. Si vous avez besoin de plus d'utilisateurs, Tina propose une variété de [plans tarifaires](https://tina.io/pricing) pour différents budgets.

## Auto-hébergé

TinaCMS étant un logiciel libre, vous pouvez de l'[auto-héberger](https://tina.io/docs/self-hosted/overview). Dans ce cas, aucuns frais n'existent d'abonnement. Vous aurez besoin d'une équipe de développement pour mettre en place votre instance de Tina et vous devrez fournir votre propre support.
