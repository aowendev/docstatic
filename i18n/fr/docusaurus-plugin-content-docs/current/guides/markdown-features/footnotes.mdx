# Notes de bas de page

Il existe une extension à Markdown qui permet d'ajouter des notes de bas de page, mais son implémentation n'est pas cohérente et sépare le contenu du numéro. Par exemple :

```markdown
Markdown est un langage de balisage léger[^1] créé par John Gruber[^2], désormais largement utilisé dans la rédaction[^3].

[^1] : Markdown utilise une syntaxe simple pour formater le texte.
[^2] : John Gruber a créé Markdown en 2004.
[^3] : De nombreuses plateformes telles que GitHub, Stack Overflow et les plateformes de blogs prennent en charge Markdown.
```

Dans DocStatic, cela serait représenté comme suit :

```markdown
Markdown est un langage de balisage léger<Footnote summary="1"> Markdown utilise une syntaxe simple pour formater le texte.</Footnote>
créé par John Gruber<Footnote summary="2">John Gruber a créé Markdown en 2004.</Footnote>,
aujourd'hui largement utilisé dans la rédaction<Footnote summary="3">De nombreuses plateformes telles que GitHub, Stack Overflow et les plateformes de blogs prennent en charge Markdown.</Footnote>.
```

Le composant **Footnotes** vous permet d'ajouter des notes de bas de page en ligne à votre texte.

1. Sélectionnez **Footnotes** dans la liste **Embed**.
2. Modifiez le composant.
3. Donnez-lui un **Résumé**.
4. Entrez la **Note de bas de page**.

#### Exemple

Markdown est un langage de balisage léger<Footnote summary="1"> Markdown utilise une syntaxe simple pour formater le texte.</Footnote> créé par John Gruber<Footnote summary="2">John Gruber a créé Markdown en 2004.</Footnote>, désormais largement utilisé dans la rédaction<Footnote summary="3">De nombreuses plateformes telles que GitHub, Stack Overflow et les plateformes de blogs prennent en charge Markdown.</Footnote>.

<Admonition type="note" title="Remarque">
  Si deux notes de bas de page ont le même contenu, elles n'apparaîtront qu'une seule fois dans le pied de page et partageront le même numéro.
</Admonition>
