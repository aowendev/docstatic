DocStatic ajoute une fonctionnalité de tableau de bord à TinaCMS. Tina génère une API GraphQL en utilisant le système de fichiers du dépôt Git comme base de données. Les tableaux de bord interrogent l'API (localement ou dans le cloud) et présentent des rapports exploitables directement dans le CMS. La fonctionnalité de tableau de bord est actuellement considérée comme bêta. La fonctionnalité a été entièrement testée, mais l'interface utilisateur est provisoire et doit être affinée. Les tableaux de bord sont mis à jour lorsque la page est actualisée. Si un autre utilisateur a apporté des modifications depuis que vous avez consulté un tableau de bord, vous pouvez cliquer sur **Actualiser** pour mettre à jour le tableau de bord.

<Admonition type="note" title="Remarque">
  En fonction de la quantité de contenu dans le système, certains tableaux de bord peuvent prendre un certain temps à se charger. Ces tableaux de bord s'actualisent généralement un peu plus rapidement lorsqu'ils sont exécutés localement.
</Admonition>

## Affichage des tableaux de bord

Dans le CMS, accédez à la collection Dashboards. En haut de la page se trouve une barre d'état qui affiche la connexion GraphQL. Elle indique si vous êtes connecté au cloud ou à l'hôte local. Si l'hôte local est disponible, il se connectera toujours à l'hôte local. Pour utiliser le cloud, assurez-vous que le serveur de développement local est arrêté. La barre d'état affiche également l'environnement et vérifie que tous les paramètres sont valides.

## Affichage de l'état du flux de travail

Le tableau de bord Aperçu du contenu affiche le nombre total de rubriques dans le chemin /docs et les rubriques dans chaque état du flux de travail :

* Brouillon
* Révision
* Traduction
* Approuvé
* Publié
* Non répertorié

Cliquez sur le lien d'état pour afficher toutes les rubriques dans cet état.

Cliquez sur Modifier pour ouvrir la rubrique à modifier.

Cliquez sur l'icône Fermer pour fermer la liste des rubriques.

### Affichage de l'activité récente

La section Activité récente affiche les rubriques récemment modifiées. Vous pouvez filtrer la période (Toutes les périodes, Semaine dernière ou Mois dernier) et définir le nombre de résultats de 5 à 50.

## Affichage de l'état de la traduction

Le tableau de bord Traductions affiche l'état de chaque rubrique pour la langue sélectionnée dans la liste :

* À jour : la date de dernière modification de la traduction est postérieure à celle de la source.
* Obsolète : la date de dernière modification de la source est postérieure à celle de la traduction.
* Manquant : il existe un fichier source sans fichier de traduction équivalent.

## Affichage de la réutilisation des médias

Le tableau de bord Bibliothèque multimédia affiche où le contenu du chemin /static/img est utilisé. Il affiche le nombre total de fichiers et d'images, la taille des fichiers, les chemins d'accès aux images et la taille des images.

Vous pouvez filtrer par :

* Tout le contenu.
* Images uniquement.
* Récents (7 jours).
* Utilisés.
* Inutilisés.

Pour le contenu utilisé, cliquez sur Utilisé dans n documents pour afficher les rubriques dans lesquelles le contenu est utilisé. Vous pouvez ensuite cliquer sur Modifier pour modifier ces rubriques.

## Vérification des liens rompus

Le tableau de bord Liens rompus valide tous les hyperliens dans les rubriques. Le rapport comprend le nombre de :

* Liens totaux.
* Liens valides pour lesquels une connexion a pu être établie.
* Liens rompus pour lesquels aucune connexion n'a pu être établie.
* Liens d'avertissement pour lesquels la connexion a été rejetée, mais le lien peut encore être valide.

Vous pouvez ignorer en toute sécurité les avertissements concernant [localhost](http://localhost) si vous décrivez un logiciel fonctionnant localement. Mais les liens vers les documents doivent être relatifs.

Les avertissements sont généralement le résultat de politiques CORS/CORP agressives qui ne respectent pas les paramètres no-cors. Vous devez valider manuellement ces sites en cliquant sur le lien hypertexte.

Le tableau de bord répertorie les fichiers contenant des liens problématiques. Vous pouvez les ouvrir en cliquant sur Modifier dans l'une ou l'autre des listes.

Le tableau de bord fournit également un résumé avec la date à laquelle le rapport a été généré.
