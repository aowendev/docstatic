Les ressources statiques sont les fichiers non codés qui sont directement copiés dans le résultat de la compilation. Elles comprennent les images, les feuilles de style, les favicons, les polices, etc.

Vous devez placer ces ressources dans le dossier « static ». Chaque fichier que vous placez dans ce dossier est copié à la racine du dossier de compilation généré, en conservant la hiérarchie des dossiers. Par exemple, si vous ajoutez un fichier nommé « sun.jpg » au dossier static, il sera copié dans « build/sun.jpg ».

Cela signifie que :

* Pour le site `baseUrl: '/'`, l'image `/static/img/sun.jpg` sera servie à l'adresse `/img/sun.jpg`.
* Pour le site `baseUrl: '/subpath/'`, l'image `/static/img/sun.jpg` sera servie à l'adresse `/subpath/img/sun.jpg`.

## Référencer votre ressource statique

### Dans le CMS

1. Insérez un lien.
2. Entrez le chemin absolu, par exemple /files/note.docx.
3. Entrez le texte à afficher.

### Dans Markdown

```markdown
Vous écrivez un lien comme ceci : [Télécharger ce document](/files/note.docx)

Docusaurus le transforme en : <a href={require('static/files/note.docx')}>Télécharger ce document</a>
```

Pour plus d'informations, consultez la section [Ressources statiques](https://docusaurus.io/docs/static-assets) dans la documentation Docusaurus.
