Les ressources statiques sont les fichiers non codés qui sont directement copiés dans la sortie de la compilation. Ils comprennent les images, les feuilles de style, les favicons, les polices de caractères, etc.

Vous devez placer ces ressources dans le dossier `static`. Chaque fichier que vous placez dans ce dossier est copié à la racine du dossier de compilation généré, en conservant la hiérarchie des dossiers. Par exemple, si vous ajoutez un fichier nommé `sun.jpg` dans le dossier statique, il sera copié dans `build/sun.jpg`.

Cela signifie que :

* Pour le site `baseUrl : '/'`, l'image `/static/img/sun.jpg` sera servie à `/img/sun.jpg`.
* Pour le site `baseUrl : '/subpath/'`, l'image `/static/img/sun.jpg` sera servie à `/subpath/img/sun.jpg`.

## Référencement de votre ressource statique

### Dans le CMS

1. Insérez un lien.
2. Saisissez le chemin absolu, par exemple `/files/note.docx`.
3. Saisissez le texte à afficher.

### En Markdown

```markdown
Vous écrivez un lien comme celui-ci : [Télécharger ce document](/files/note.docx)

Docusaurus le remplace par : <a href={require('static/files/note.docx')}>Télécharger ce document</a>
```

Pour plus d'informations, voir [Ressources statiques](https://docusaurus.io/fr/docs/static-assets) dans la documentation de Docusaurus.
