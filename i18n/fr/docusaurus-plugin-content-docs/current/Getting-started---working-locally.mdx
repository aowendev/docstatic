# Premiers pas pour travailler avec DocStatic en local

Bienvenue ! Ce guide vous explique comment configurer DocStatic pour la première fois.

Si vous n'avez jamais utilisé GitHub ou docs-as-code auparavant, ne vous inquiétez pas : nous vous expliquerons ce dont vous avez besoin, pourquoi et où trouver plus d'informations.

## Prérequis

Vous pouvez exécuter DocStatic localement (sur votre ordinateur) ou l'héberger dans le cloud. Nous vous expliquons ici comment le configurer pour qu'il fonctionne localement.

Avant de commencer, vous aurez besoin de :

* [GitHub ](https://github.com/) — créez un compte gratuit. GitHub stockera votre contenu et suivra les modifications.
* Un environnement de développement intégré (IDE) ou un éditeur de texte. Voici deux options open source :
  * [Eclipse](https://eclipseide.org)
  * [VSCodium](https://vscodium.com/)

Une fois que vous disposez de ces éléments, vous pouvez commencer l'installation (comme décrit dans cet article).

***

## Installer Node.js

Vous devez installer la version 18.0 ou supérieure de [Node.js](https://nodejs.org/), car DocStatic utilise ses outils de compilation.

Commencez par vérifier si Node.js est déjà installé :

1. Ouvrez un terminal (invite de commande ou application Terminal).
2. Exécutez :

```
node -v
```

Si vous voyez un numéro de version commençant par **v18 ou supérieur**, vous disposez déjà d'une version appropriée de Node.js.

Si Node.js n'est **pas installé** ou si vous disposez d'une **version plus ancienne** :

1. Accédez à la page [de téléchargement officielle de Node.js](https://nodejs.org/en/download/prebuilt-installer).
2. Choisissez la version LTS (Long-Term Support) pour votre système d'exploitation.
   Le programme d'installation comprend npm, le gestionnaire de paquets Node, dont vous aurez besoin dans les étapes suivantes.
3. Pendant l'installation :
   * Conservez les options par défaut sélectionnées (elles incluent les dépendances requises)
   * Une fois l'installation terminée, fermez et rouvrez votre terminal.
4. (Facultatif) si vous travaillez sur différents projets qui nécessitent différentes versions de Node, installez nvm ([Node Version Manager](https://github.com/nvm-sh/nvm)). Avec nvm, vous pouvez passer d'une version de Node à l'autre sans perturber votre configuration.

Suivant : Installez Yarn.

***

## Installer Yarn

DocStatic utilise Yarn pour la gestion des paquets.

Vérifiez si Yarn est déjà installé.

1. Ouvrez votre terminal ou votre invite de commande.
2. Tapez :

```
yarn -v
```

Si vous voyez un numéro de version, Yarn est déjà installé.

Si vous obtenez un message « commande introuvable » ou une erreur similaire :

1. Ouvrez un terminal (invite de commande ou application Terminal).
2. Exécutez :

```
npm install -g yarn
```

<Admonition type="note" title="Remarque">
  Sous macOS ou Linux, vous devrez peut-être préfixer la commande avec sudo pour accorder l'autorisation :

  ```
  sudo npm install -g yarn
  ```
</Admonition>

<Admonition type="note" title="Remarque">
  Sous Windows, vous n'utilisez pas sudo. Il vous suffit d'ouvrir votre terminal en tant qu'administrateur et d'exécuter la commande.
</Admonition>

Suivant : Forker le dépôt DocStatic

***

## Dupliquez le dépôt DocStatic

Le code source de DocStatic se trouve sur GitHub. Pour commencer, vous devez dupliquer le dépôt.

Lorsque vous dupliquez un dépôt, vous créez votre propre copie personnelle de celui-ci dans GitHub. Comme il s'agit de votre propre copie, vous disposez des droits de gestion et pouvez y ajouter du contenu. C'est ainsi que vous pouvez travailler sur un dépôt sans affecter le dépôt DocStatic d'origine.

1. Connectez-vous à votre compte GitHub.
2. Rendez-vous sur [https://github.com/aowendev/docstatic](https://github.com/aowendev/docstatic).
3. Cliquez sur **Fork** (généralement en haut à droite).
4. Laissez le **nom du référentiel** par défaut.
5. Entrez une description de votre projet.
6. Assurez-vous que l'option **Copier uniquement la branche principale** est cochée.
7. Cliquez sur **Créer une fourche**.

GitHub crée une nouvelle branche du référentiel dans votre compte GitHub.

Étape suivante : cloner votre fourche.

***

## Cloner votre fourche du référentiel DocStatic

Le clonage vous permet de travailler localement sur le contenu que vous allez pousser vers votre fourche sur GitHub.

* Fork = propriété sur GitHub
* Clone = possibilité de travailler localement

Pour créer un clone :

1. Connectez-vous à votre compte GitHub.
2. Accédez au référentiel du fork.
3. Cliquez sur le référentiel pour l'ouvrir.
4. Cliquez sur **\<> Code**, puis copiez l'URL.
5. Ouvrez votre terminal ou votre invite de commande.
6. Utilisez `cd` pour accéder à l'emplacement où vous souhaitez stocker le clone. Par exemple, `cd Documents/Projets/`.
7. Entrez `git clone` suivi de l'URL de votre clone.

```shell
git clone https://github.com/acme-projects/docstatic.git
```

Suivant : Installez les dépendances du projet.

***

## Installez les dépendances du projet

Pour travailler localement, vous devez également installer les dépendances du projet.

1. Ouvrez votre terminal ou votre invite de commande.
2. Utilisez `cd` pour naviguer jusqu'au dossier racine de votre dépôt DocStatic cloné ou forké et installez les paquets :

```
cd docstatic
yarn install
```

Yarn téléchargera tout ce qui est répertorié dans package.json. Cela peut prendre quelques minutes.

Vous avez maintenant tout configuré. Passons maintenant à la structure du projet.

***

## Structure du projet

Après avoir forké et cloné le dépôt, vous verrez divers fichiers dans votre dossier `docstatic/`. Ci-dessous, nous avons inclus **certains** des fichiers et dossiers de la structure du projet que vous devez connaître. **Il ne s'agit pas d'une liste exhaustive de tout ce que contient le projet**.

```
│   ├── homepage

│   └── sidebar
│       └── index.json
│
├── docs

├── sidebars.ts

```

### Aperçu de la structure du projet

* `/apis/` - Fichiers YAML OpenAPI.
* `/blog/` - Fichiers MDX du blog.
* `/config/` - Fichiers JSON utilisés par TinaCMS pour configurer DocStatic.
* `/docs/` - Fichiers MDX de la documentation.
* `/i18n/` - Fichiers de traduction.
* `/reuse/` - Contenu réutilisable.
* `/src/` - Fichiers non liés à la documentation, tels que des pages ou des composants React personnalisés.
  * `/src/pages` - Tous les fichiers JSX/TSX/MDX contenus dans ce répertoire sont convertis en pages web.
* `/static/` - Dossier statique. Tout contenu présent ici est copié à la racine du dossier de compilation final.
* `/docusaurus.config.ts` - Fichier de configuration contenant la configuration du site.
* `/package.json` - Un site web DocStatic est une application React. Vous pouvez installer et utiliser tous les paquets npm que vous souhaitez.
* `/sidebars.ts` - Spécifie l'ordre des documents dans la barre latérale. Utilisez-le pour votre structure de « table des matières ».

Ensuite, voyons où vous devrez créer et stocker vos fichiers de documentation.

***

## Effacer le dossier Docs

Le dossier Docs contient une copie de la documentation DocStatic. L'idée est d'effacer la documentation DocStatic. Lorsque vous créez votre propre documentation, vous devez l'enregistrer dans le dossier Docs.

***

## Monorepos

DocStatic permet d'utiliser un seul dépôt contenant à la fois le code du projet et la documentation du projet. Dans la terminologie Docusaurus, ce concept est appelé « monorepo ».

Pour plus d'informations, consultez [Monorepos](https://docusaurus.io/docs/installation#monorepos) dans la documentation Docusaurus.

***

## Prévisualiser les modifications

Pour prévisualiser vos modifications au fur et à mesure que vous modifiez les fichiers, vous pouvez exécuter un **serveur de développement local** qui servira votre site web et reflétera les dernières modifications.

Ouvrez un terminal ou une invite de commande et entrez :

```shell
yarn dev
```

Par défaut, une fenêtre de navigateur s'ouvrira à l'adresse [http://localhost:3000](http://localhost:3000).

***

## Compilation

DocStatic utilise un générateur de site statique pour compiler le site web dans un dossier de contenu statique et le placer sur un serveur web où il peut être consulté. Pour compiler le site web, utilisez :

```shell
yarn build-local
```

Le contenu est généré dans le dossier `/build`, que vous pouvez copier vers n'importe quel service d'hébergement de fichiers statiques tel que GitHub pages, Netlify ou Vercel. Pour plus d'informations, consultez la section [Déploiement](https://docusaurus.io/docs/deployment) dans la documentation Docusaurus.
