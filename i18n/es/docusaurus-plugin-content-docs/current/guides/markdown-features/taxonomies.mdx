# Etiquetas taxonómicas

Las etiquetas taxonómicas te permiten aplicar una [arquitectura de la información](https://en.wikipedia.org/wiki/Information_architecture) al contenido de tu sitio. Deben aplicarse a cualquier plataforma de documentación que utilices, como los sistemas de seguimiento de errores y las wikis. Los requisitos de etiquetas más restrictivos con los que probablemente te encontrarás son los sistemas planos (no jerárquicos) que solo admiten letras minúsculas y guiones bajos (`_`) sin espacios.

DocStatic aplica una jerarquía de tres niveles a las etiquetas de taxonomía. Para evitar que las etiquetas se vuelvan excesivamente largas, el anidamiento se limita a una profundidad máxima de tres. En algunos casos, esto requiere aplanar la estructura (por ejemplo, aunque «regions*europe*uk*scotland» es lógico, tiene cuatro niveles, por lo que debe estructurarse como «regions*uk\_scotland»).

Al diseñar una arquitectura de información de tres niveles, se recomienda que intente tener alrededor de diez categorías de nivel superior, siete categorías de nivel medio y tantas categorías de nivel inferior como necesite. Debe escribir las siglas y abreviaturas y tratar de evitar la redundancia.

## Edición de etiquetas de taxonomía

1. Desde la colección **Taxonomías**, **Añada** una taxonomía.
2. Asigne una **Etiqueta** utilizando solo letras minúsculas.
3. Añada cualquier **Hijo** y asígnele una etiqueta, comenzando por la etiqueta principal seguida de un guión bajo.
4. Para cada subcategoría, añada las subcategorías que desee.

<Admonition type="note" title="Nota">
  Puede introducir etiquetas sin el nombre de la etiqueta precedente y el guión bajo, pero si lo hace al añadirlas a un tema, se mostrarán como una lista plana.
</Admonition>
