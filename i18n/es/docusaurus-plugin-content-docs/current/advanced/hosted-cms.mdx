# CMS alojado

Uno de los dos componentes principales de DocStatic es el CMS (el otro es el SSG). Hasta ahora, lo más probable es que solo hayas utilizado la versión local, invocada desde la línea de comandos con `yarn dev`. Pero DocStatic también admite dos formas de utilizar una versión alojada del CMS.

## TinaCloud

La opción más sencilla de CMS alojado es [TinaCloud](https://tina.io/docs/tinacloud). Puede [registrarse](https://app.tina.io/register) para obtener una cuenta gratuita, conectarla a su repositorio de GitHub y empezar a utilizarla inmediatamente. El nivel gratuito admite hasta dos usuarios, lo que puede ser suficiente para escritores independientes y pequeños equipos de documentación. Si necesita más usuarios, Tina ofrece una variedad de [planes de precios](https://tina.io/pricing) para diferentes presupuestos.

## Autoalojado

Dado que TinaCMS es de código abierto, tienes la opción de [autoalojarlo](https://tina.io/docs/self-hosted/overview). En este caso, no hay cuotas de suscripción, pero necesitarás un equipo de desarrollo para configurar tu instancia de Tina y tendrás que proporcionar tu propio soporte técnico.
