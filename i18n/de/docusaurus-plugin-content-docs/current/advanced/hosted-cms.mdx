# Gehostetes CMS

Eine der beiden Kernkomponenten von docStatic ist das CMS (die andere ist das SSG). Bislang haben Sie wahrscheinlich nur die lokale Version verwendet, die über die Befehlszeile mit „yarn dev“ aufgerufen wird. docStatic unterstützt jedoch auch zwei Möglichkeiten, eine gehostete Version des CMS zu verwenden.

## TinaCloud

Die einfachste Option für ein gehostetes CMS ist [TinaCloud](https://tina.io/docs/tinacloud). Sie können sich für ein kostenloses Konto [registrieren](https://app.tina.io/register), es mit Ihrem GitHub-Repository verbinden und sofort loslegen. Die kostenlose Version unterstützt bis zu zwei Benutzer, was für einzelne Autoren und kleine Dokumentationsteams ausreichend sein dürfte. Wenn Sie mehr Benutzer benötigen, bietet Tina eine Vielzahl von [Preisplänen](https://tina.io/pricing) für unterschiedliche Budgets an.

## Selbst gehostet

Da TinaCMS Open Source ist, haben Sie die Möglichkeit, es selbst zu hosten (https\://tina.io/docs/self-hosted/overview). In diesem Fall fallen keine Abonnementgebühren an, aber Sie benötigen ein Entwicklungsteam, um Ihre Tina-Instanz einzurichten, und müssen selbst für den Support sorgen.
