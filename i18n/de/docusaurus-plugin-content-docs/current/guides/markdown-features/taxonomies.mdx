# Taxonomie-Tags

Mit Taxonomie-Tags können Sie eine [Informationsarchitektur](https://en.wikipedia.org/wiki/Information_architecture) auf Ihre Website-Inhalte anwenden. Diese sollte auf alle von Ihnen verwendeten Dokumentationsplattformen angewendet werden, beispielsweise Bug-Tracking-Systeme und Wikis. Die restriktivsten Tag-Anforderungen, denen Sie wahrscheinlich begegnen werden, sind flache (nicht-hierarchische) Systeme, die nur Kleinbuchstaben und Unterstriche (`_`) ohne Leerzeichen unterstützen.

DocStatic wendet eine dreistufige Hierarchie auf Taxonomie-Tags an. Um zu verhindern, dass die Tags unangemessen lang werden, ist die Verschachtelung auf eine maximale Tiefe von drei Ebenen begrenzt. In einigen Fällen erfordert dies eine Vereinfachung der Struktur (zum Beispiel ist „regions*europe*uk*scotland“ zwar logisch, hat aber vier Ebenen und muss daher als „regions*uk\_scotland“ strukturiert werden).

Bei der Gestaltung einer dreistufigen Informationsarchitektur wird empfohlen, etwa zehn Kategorien der obersten Ebene, sieben Kategorien der mittleren Ebene und so viele Kategorien der untersten Ebene wie nötig anzustreben. Sie sollten Akronyme und Abkürzungen ausschreiben und Redundanzen vermeiden.

## Taxonomie-Tags bearbeiten

1. Fügen Sie aus der Sammlung **Taxonomien** eine Taxonomie **hinzu**.
2. Geben Sie ihr einen **Tag**, der nur Kleinbuchstaben enthält.
3. Fügen Sie beliebige **Unterelemente** hinzu und geben Sie ihnen einen Tag, der mit dem übergeordneten Tag beginnt, gefolgt von einem Unterstrich.
4. Fügen Sie für jedes untergeordnete Element weitere untergeordnete Elemente hinzu.

<Admonition type="note" title="Hinweis">
  Sie können Tags ohne den vorangestellten Tag-Namen und Unterstrich eingeben, aber wenn Sie dies beim Hinzufügen zu einem Thema tun, werden sie als flache Liste dargestellt.
</Admonition>
